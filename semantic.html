What Is Semantic HTML?

A semantic element clearly describes its meaning to both the browser and the developer, instead of just being a generic box. Before HTML5, most page layouts relied on plain <div> elements; semantic tags replace many of those with names that say what the content actually is.

Common Semantic Elements

  • <header> — introductory content or navigation for a page or section
  • <nav> — a block of navigation links (used at the top of every page on this site)
  • <main> — the primary content of the page
  • <article> — a self-contained piece of content, like a blog post
  • <section> — a themed grouping of content
  • <aside> — content related to, but separate from, the main content
  • <footer> — closing content for a page or section
<header>
    <nav>...</nav>
</header>
<main>
    <article>...</article>
</main>
<footer>...</footer>

Why It Matters

Semantic markup makes a page easier for search engines to index and easier for screen readers to navigate, which connects this topic directly to Accessibility. It also makes source code easier for other developers to read.
