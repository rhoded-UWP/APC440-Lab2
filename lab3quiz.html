Question 2

In CSS, what is the process called when a style rule applied to a parent element is automatically passed down to a child element?

Choose one answer:

Correct. The child inherits the parent's value, but a rule that targets the child directly takes precedence over anything inherited. Inheritance supplies a starting value; it does not lock it in.

Not quite — this has the precedence backwards. The name is right, but a rule targeting the child directly beats the value handed down from the parent, not the other way around.

Not quite. !important is a real part of CSS, but it is not needed here. An ordinary rule on the child already overrides an inherited value, because a direct rule outranks inheritance on its own.

Not quite, though this is close. Inheritance still happens even when the child has its own rules — the inherited value is simply overridden for that one property. The process is not switched off; it is outranked.
